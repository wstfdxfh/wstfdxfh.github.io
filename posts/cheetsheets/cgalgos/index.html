一个点是否在三角形内部#

  • 可以使用叉乘的方法判断一个点是否在三角形内部。当三角形中所有边的对点和点P都位于该边的一侧时，就说明P在三角形内部，而判断是否位于一侧可以使用叉乘的符号是否相等来判断。

#include <iostream>

struct Point {
  double x, y;
};

// 向量叉积：计算向量a × 向量b的值
double crossProduct(const Point& a, const Point& b) {
  return a.x * b.y - a.y * b.x;
}

// 检查点p1和p2是否在边a-b的同一侧（包含边上的点）
bool isSameSide(const Point& p1, const Point& p2, const Point& a,
                const Point& b) {
  Point ab = {b.x - a.x, b.y - a.y};  // 边a→b的向量
  Point ap1 = {p1.x - a.x, p1.y - a.y};
  Point ap2 = {p2.x - a.x, p2.y - a.y};
  double cp1 = crossProduct(ab, ap1);  // 计算叉积AB × AP1
  double cp2 = crossProduct(ab, ap2);  // 计算叉积AB × AP2
  return (cp1 * cp2) >= 0;             // 符号相同或其中一个为0（点在线段上）
}

// 判断点P是否在三角形ABC内
bool isInsideTriangle(const Point& A, const Point& B, const Point& C,
                      const Point& P) {
  return isSameSide(P, C, A, B) &&  // P和C在AB同侧
         isSameSide(P, A, B, C) &&  // P和A在BC同侧
         isSameSide(P, B, C, A);    // P和B在CA同侧
}

int main() {
  Point A = {0, 0};
  Point B = {3, 0};
  Point C = {1.5, 2.5};
  Point P = {1.5, 10};  // 待检测点

  if (isInsideTriangle(A, B, C, P)) {
    std::cout << "点P在三角形内部" << std::endl;
  } else {
    std::cout << "点P在三角形外部" << std::endl;
  }
  return 0;
}

欧拉路径#

  1. 问题本质

题目要求“是否存在一条路径恰好经过所有有向边一次”，其实就是有向欧拉路径 (Eulerian Path) / 欧拉回路 (Eulerian Circuit) 的判定与构造问题。

  • 欧拉路径：一条路径，经过每一条边且恰好一次。
  • 欧拉回路：欧拉路径同时是一个环，即起点 = 终点。
  1. 判定条件（有向图）

对于一个有向图存在欧拉路径，需要满足以下条件：

  1. 度条件
    • 至多一个点满足：出度 = 入度 + 1 → 可能的起点。
    • 至多一个点满足：入度 = 出度 + 1 → 可能的终点。
    • 其他所有点必须满足：入度 = 出度。
    • 如果所有点都是 入度 = 出度，那说明存在欧拉回路。
  2. 连通性条件
    • 把所有边当作无向边忽略方向，所有度 > 0 的节点必须在同一个连通分量中。否则不可能走完所有边。

这两个条件必须同时成立。

  1. 构造方法（Hierholzer 算法）

在判定条件成立后，我们需要输出路径。这里用 Hierholzer 算法：

  • 原理：
    1. 从起点开始走，每次选择一条未走过的边前进，直到走不动为止（得到一条子回路）。
    2. 如果图中还有没走过的边，就在当前路径中找一个还有出边未用完的节点，从那里继续走，形成新的子回路。
    3. 不断拼接这些子回路，最终形成完整的欧拉路径。
  • 时间复杂度：O(n + m)，只遍历所有边一次。
  1. 最终输出

  • 如果条件不满足 → 输出 "no path"。

  • 如果条件满足 → 输出一条长度为 m+1 的节点序列（因为 m 条边对应 m+1 个节点）。

两条线段是否相交#

数学方法

把起点—终点记为线段 $AB$，墙记为线段 $CD$。

设二维叉积

$$ \operatorname{cross}(\vec{u},\vec{v}) = u_x v_y - u_y v_x. $$

定义三点 $P,Q,R$ 的有向面积（朝向）：

$$ \operatorname{orient}(P,Q,R)=\operatorname{cross}(Q-P,;R-P). $$

线段相交（含重合/端点接触）充要条件 两线段 $AB$ 与 $CD$ 相交当且仅当：

  1. $\operatorname{orient}(A,B,C)$ 与 $\operatorname{orient}(A,B,D)$ 异号或为 0，且
  2. $\operatorname{orient}(C,D,A)$ 与 $\operatorname{orient}(C,D,B)$ 异号或为 0，
  3. 共线（orient=0）时，再检查投影是否在各自的包围盒内有重叠。

用符号表示就是：

$$ \begin{aligned} &\text{let } o_1=\operatorname{orient}(A,B,C),; o_2=\operatorname{orient}(A,B,D),\ &\qquad o_3=\operatorname{orient}(C,D,A),; o_4=\operatorname{orient}(C,D,B).\ &\text{若 } \operatorname{sgn}(o_1)\cdot\operatorname{sgn}(o_2)\le 0 \text{ 且 } \operatorname{sgn}(o_3)\cdot\operatorname{sgn}(o_4)\le 0,\ &\text{并且在任一 } o_i=0 \text{ 的情况下满足包围盒重叠，则两段相交。} \end{aligned} $$

“重合也算挡住”，因此只要判定为相交（包括端点接触与部分/全部重合）就视为被挡住。 C++ 简洁实现

#include <bits/stdc++.h>
using namespace std;

struct Pt { double x, y; };

static const double EPS = 1e-9;

int sgn(double x) { return (x > EPS) - (x < -EPS); }

double cross(const Pt& a, const Pt& b, const Pt& c) {
    // cross((b-a), (c-a))
    return (b.x - a.x) * (c.y - a.y) - (b.y - a.y) * (c.x - a.x);
}

bool onSegment(const Pt& a, const Pt& b, const Pt& p) {
    // p 与 ab 共线且落在包围盒内
    if (sgn(cross(a, b, p)) != 0) return false;
    return (min(a.x, b.x) - EPS <= p.x && p.x <= max(a.x, b.x) + EPS &&
            min(a.y, b.y) - EPS <= p.y && p.y <= max(a.y, b.y) + EPS);
}

bool segIntersect(const Pt& a, const Pt& b, const Pt& c, const Pt& d) {
    // 处理退化：若墙或路径为点
    bool ab_point = (sgn(a.x - b.x) == 0 && sgn(a.y - b.y) == 0);
    bool cd_point = (sgn(c.x - d.x) == 0 && sgn(c.y - d.y) == 0);
    if (ab_point && cd_point) return (sgn(a.x - c.x) == 0 && sgn(a.y - c.y) == 0);
    if (ab_point) return onSegment(c, d, a);
    if (cd_point) return onSegment(a, b, c);

    double o1 = cross(a, b, c);
    double o2 = cross(a, b, d);
    double o3 = cross(c, d, a);
    double o4 = cross(c, d, b);

    int s1 = sgn(o1), s2 = sgn(o2), s3 = sgn(o3), s4 = sgn(o4);

    if (s1 == 0 && onSegment(a, b, c)) return true;
    if (s2 == 0 && onSegment(a, b, d)) return true;
    if (s3 == 0 && onSegment(c, d, a)) return true;
    if (s4 == 0 && onSegment(c, d, b)) return true;

    return (s1 * s2 < 0) && (s3 * s4 < 0);
}

// 是否被墙挡住（含重合/接触）
bool blocked(double xs, double ys, double xe, double ye,
             double x1, double y1, double x2, double y2) {
    Pt A{xs, ys}, B{xe, ye}, C{x1, y1}, D{x2, y2};
    return segIntersect(A, B, C, D);
}

// 示例
int main() {
    // 起点-终点
    double xs = 0, ys = 0, xe = 4, ye = 0;
    // 墙：横跨 (2,-1) 到 (2,1)，显然与路径相交
    double x1 = 2, y1 = -1, x2 = 2, y2 = 1;

    cout << (blocked(xs, ys, xe, ye, x1, y1, x2, y2) ? "BLOCKED" : "CLEAR") << "\n";

    // 一个不相交的例子
    cout << (blocked(0,0,4,0, 5,-1,6,1) ? "BLOCKED" : "CLEAR") << "\n";

    // 重合的例子（共线且重叠）
    cout << (blocked(0,0,4,0, 1,0,3,0) ? "BLOCKED" : "CLEAR") << "\n";
    return 0;
}

说明

  • EPS 处理浮点误差；sgn 给出三态比较结果。
  • cross(a,b,c) 是 $\operatorname{orient}(a,b,c)$。
  • onSegment 在共线前提下检查是否落在包围盒里。
  • segIntersect 覆盖了普通相交、端点接触、共线重叠，以及退化为点的情况。
  • blocked 即为“从起点到终点是否被墙挡住”的答案函数。

三角形内部随机采样#

问题描述：假设你有一个二维平面上的三角形，由三个顶点A(x1, y1)、B(x2, y2)、C(x3, y3)组成。请设计一种方法，在这个三角形内部随机均匀地采样一个点。

正确答案： 随机均匀地在一个三角形内采样一个点的方法之一是使用重心坐标法（Barycentric Coordinates）。具体步骤如下：

  1. 生成两个在[0,1]区间内的随机数u和v。
  2. 如果u + v > 1，则将其映射到三角形内部：令u = 1 - u，v = 1 - v。
  3. 计算第三个参数w = 1 - u - v。
  4. 最终采样点P的坐标为：P = uA + vB + w*C。
